# 01 项目总览

## 1.1 项目名称

SaucyClaw

## 1.2 项目目标

SaucyClaw 用于展示一套围绕 OpenClaw 的多智能体组织化治理样例，重点解决以下常见问题：

1. 收到任务后多个 agent 同时启动
2. 没有唯一流程负责人
3. specialist 直接面向用户并发交付
4. 没有 handoff、review、状态板
5. 角色边界不清、流程无法复用

## 1.3 设计原则

### 原则一：General Manager 先分流，后决定是否调人
不是所有任务都需要多智能体。  
General Manager 必须先判断任务处理模式，再决定是否自己做、只调用一个 specialist、或升级为多角色协作。

### 原则二：多智能体是升级动作，不是默认动作
只有在单人或单 specialist 无法稳定闭环时，才允许升级为多角色串行协作。

### 原则三：复杂任务必须有交接与评审
复杂任务未经 review 不得视为 done。  
每次正式流转必须有 handoff。

### 原则四：prompt / soul / memory 分层
- prompt：当前执行逻辑
- soul：角色本质与长期边界
- memory：经验、偏好、常见错误

### 原则五：先改文件设定，再谈对话记忆
长期有效的行为必须写入文件，而不是依赖上下文中的“请记住”。

## 1.4 项目产出

本项目包含：

- 完整目录结构
- 系统级治理文件
- 角色设定文件
- specialist 设定
- 模板与示例
- 可直接复制的 OpenClaw 组织样例

## 1.5 推荐使用方式

建议把本项目当作：

- 组织规范样板
- agent 角色模板库
- 多智能体治理方法展示仓库
- 对外分享和交流的案例工程
