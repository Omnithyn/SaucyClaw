# SaucyClaw Hardness Roadmap

## 当前阶段

SaucyClaw 已具备：

- General Manager 主导的调度思想
- 基本的角色边界
- Codex 可进入的入口层
- 初步的评测与 rubric 资产

## 下一阶段目标

### 1. 项目身份升级
把项目从 OpenClaw 多智能体治理样板，升级为更通用的 agent hardness engineering 基线。

### 2. 角色体系升级
在现有总控、specialist、reviewer 之外，逐步补齐：
- evaluator
- policy guardian

### 3. 评测体系升级
逐步完善：
- smoke tasks
- replay scenarios
- rubrics
- reports

### 4. 模板体系升级
补强：
- task contract
- handoff
- review output
- evaluation report

### 5. 可迁移性升级
让 SaucyClaw 不只适配 OpenClaw，也能更自然地映射到 Codex、Claude 和其他 agent 工作流。

## 推荐推进顺序

1. 补 system 级 hardness 文档
2. 补 evals/ 目录资产
3. 补模板与验证脚本
4. 再逐步重写 README 与系统总规
