# ADR-0001: Phase 0-1 范围冻结

> 日期：2026-04-12
> 状态：已接受

---

## 背景

SaucyClaw 从治理模板项目演进为可进化的多智能体治理 Harness。
目标态架构（三域架构、本体五层、证据图、评估闭环）已经定义，
但从当前态到目标态之间需要一条清晰的迁移路径。

## 决策

Phase 0-1 仅实现以下七个模块：

- `core/meta_model` — 元模型定义与校验
- `core/governance` — 治理规则与决策逻辑
- `core/events` — 宿主事件标准化（纯转换层）
- `core/evidence` — 证据模型与生成
- `core/engine` — 编排闭环
- `adapters/openclaw` — 首个宿主适配
- `stores/file` — 本地文件存储实现

目标态架构保留在 `docs/superpowers/specs/` 中作为长期方向，
但 Phase 0-1 不实现目标态的全部能力。

## 取舍

### 选择小范围的理由

1. 当前仓库没有 Python 核心模块（tools/ 是脚本包装层）
2. 从 0 到 1 需要先跑通最小闭环，再扩展
3. 避免引入重依赖（数据库、Web 框架）改变项目性质

### 代价

1. Phase 0-1 不验证目标态架构的可行性
2. Provider 抽象层只做最小接口，不产品化
3. 评估和进化只做 L1-L2，不做自动优化

## 后果

- Phase 0-1 完成后，SaucyClaw 具备可测试的最小治理内核
- 后续阶段可以在接口骨架上逐步填充 Provider、图、多宿主
- 接口设计需保证向后兼容，避免后续重构

## 暂不做

- Web 管理台
- 图数据库
- 独立 sidecar 进程
- 复杂 DSL / 解析器
- 多宿主并行适配
- 自研复杂评估算法
- L3/L4 进化
- 实时流式事件总线
