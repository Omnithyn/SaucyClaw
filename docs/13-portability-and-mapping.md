# 可迁移性与映射说明

## 1. 目标

SaucyClaw 需要具备方法可迁移、结构可映射、入口可适配的特征。

这意味着它不应只在某一个 agent 工具里成立，而应能够在不同工作方式中保持核心治理思想。

## 2. 哪些内容应保持稳定

以下内容应尽量跨工具保持稳定：

- General Manager 先判断再调度
- specialist 默认待命
- reviewer 只审不代写
- handoff 留痕
- 单任务单 owner
- 评测样例与 rubric

## 3. 哪些内容可以因工具而变化

以下内容可以根据具体工具适配：

- 入口文件名称
- 角色文件的扫描方式
- 工具调用方式
- 运行脚本形式
- 工作区目录风格

## 4. OpenClaw 映射

在 OpenClaw 里，SaucyClaw 更像：

- 一套组织化工作区模板
- 一套角色文件与系统规则集合
- 一套便于复制与扩展的团队样板

## 5. Codex 映射

在 Codex 里，SaucyClaw 更像：

- 一个可直接进入的治理型仓库
- 一个带有 smoke tasks 与验证脚本的迭代基线
- 一个适合做 agent 架构评测的项目空间

## 6. Claude 类工作方式映射

在 Claude 类工作方式中，SaucyClaw 更像：

- 团队角色与系统规则的文件化基线
- 便于多角色长期设定与阶段流转的工程框架

## 7. 结论

SaucyClaw 的长期价值，不在于绑定某个工具，而在于把强团队、强约束、强评测的方法沉淀为可迁移的工程资产。
