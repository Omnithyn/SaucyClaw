# 项目定位升级说明

## 1. 新定位

SaucyClaw 不再只被定义为 OpenClaw 的多智能体治理样例。

更准确的定位应是：

- 通用 agent hardness engineering 基线
- 强调角色边界、调度纪律、评测与回归的团队工程仓库
- 同时兼容 OpenClaw、Codex、Claude 等不同 agent 工作方式的迁移底座

## 2. 为什么要升级定位

如果只停留在 OpenClaw 骨架层，项目容易局限在：

- 只展示目录与角色分工
- 只强调 prompt 与文件设定
- 缺少评测、回放、rubric 与回归能力

而 hardness engineering 需要更完整的能力：

- 角色要强，但不能越界
- 团队能协作，但不能失控
- 结果能交付，也能回放和比较

## 3. 迁移重点

项目升级的重点不在于抛弃 OpenClaw，而在于：

1. 保留现有治理思想
2. 增加约束层与评测层
3. 增加与多种 agent 工具链的兼容入口
4. 让 SaucyClaw 更适合作为长期演进的团队工程基线

## 4. 现阶段推进顺序

1. 先补 hardness engineering 资产
2. 再逐步重写系统主文档
3. 再逐步重写 README 与项目总览
