# ORCHESTRATION.md - 运行编排

## 一、起点

所有任务以 General Manager 的“任务处理判断”作为正式起点。

若未完成“任务处理判断”，不得启动任何 specialist。

## 二、标准流程

### A 类
1. General Manager 判断为 A 类
2. General Manager 直接执行
3. General Manager 直接对外交付

### B 类
1. General Manager 判断为 B 类
2. 点名一个 specialist
3. specialist 交付中间产物
4. General Manager 收口并对外输出

### C 类
1. General Manager 判断为 C 类
2. 点名一个 specialist
3. specialist 交付中间产物
4. reviewer 审查
5. General Manager 收口并对外输出

### D 类
1. General Manager 判断为 D 类
2. General Manager 拆解阶段
3. 只启动第一阶段 owner
4. owner 交付阶段产物
5. 写 handoff
6. reviewer（如需要）审查
7. General Manager 决定是否进入下一阶段
8. 全部阶段完成后统一收口

### E 类
1. General Manager 判断为 E 类
2. 不启动 specialist
3. 先整理缺口或风险
4. 视情况升级给 CEO 或等待条件补足

## 三、默认等待机制

任何 specialist 若未被明确点名，则默认待命，禁止自行启动。

## 四、单任务单 owner

每个阶段只能有一个 owner。  
允许输入协作，不允许多人共同直接背责。
