# STANDARDS.md - 输出与评审标准

## 一、通用要求

所有阶段产物都应尽量满足：

1. 目标明确
2. 范围清晰
3. 结构清楚
4. 结论明确
5. 可执行
6. 可验收
7. 可追踪
8. 不越界

## 二、specialist 输出要求

specialist 默认输出中间产物，不默认直接面向用户形成最终答案。

中间产物建议说明：
- 解决了什么
- 还没解决什么
- 下一步建议是什么

## 三、handoff 要求

至少包含：
- From
- To
- Task ID
- Summary
- Delivered Items
- Risks / Open Questions
- Recommended Next Action

## 四、review 要求

reviewer 只能输出：
- pass
- revise
- reject

并必须说明：
- 审查依据
- 发现的问题
- 修改要求

## 五、evaluation 要求

evaluator 的输出至少应包含：
- 评测对象
- 使用的样例或场景
- 观察结果
- 评分或结论
- 后续建议

## 六、policy check 要求

policy guardian 的输出至少应包含：
- 检查对象
- 发现的边界问题
- 影响说明
- 建议动作

## 七、General Manager 收口要求

最终对外交付前，General Manager 应确认：

1. 当前结果是否已完成必要流转
2. 是否仍有未解决的边界风险
3. 是否需要附带评测或 review 结论
4. 是否存在不应被包装为最终结论的中间内容
