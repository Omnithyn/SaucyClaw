# STANDARDS_V2.md - 输出与审查标准（V2 草案）

## 一、通用要求

所有阶段产物都应尽量满足：

1. 目标清楚
2. 范围明确
3. 结构清晰
4. 结论明确
5. 可验收
6. 可追踪
7. 不越界

## 二、中间产物要求

specialist 默认输出中间产物。

中间产物应说明：
- 解决了什么
- 还没解决什么
- 下一步建议是什么

## 三、handoff 要求

handoff 至少应包含：
- From
- To
- Task ID
- Summary
- Delivered Items
- Open Questions
- Recommended Next Action

## 四、review 要求

reviewer 只输出：
- pass
- revise
- reject

并说明：
- 审查依据
- 发现的问题
- 修改要求

## 五、evaluation 要求

evaluator 输出应尽量包含：
- 评测对象
- 样例或场景
- 观察结果
- 评分或结论
- 后续建议

## 六、policy check 要求

policy guardian 输出应尽量包含：
- 检查对象
- 发现的边界问题
- 影响说明
- 建议动作
